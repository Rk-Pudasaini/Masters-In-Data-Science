#2. Locate median of mpg variable of mtcars data graphically and compare the median value with in-built function of R

First Creating the histogram and locating the median with a line

hist(df$mpg , main = "Histogram of mpg Variable", xlab = "mpg", ylab = "Frequency")

We can also locate the median graphically for that we need to calculate the more than Cumulative sum and Less Than of Mpg variable and by plotting them in the same plot we get the intersection and that intersection will represent the median value

#this can be done graphically as well 
mpg_sorted <- sort(df$mpg)

# Calculate the cumulative sum of frequencies for the "mpg" data
freq_more_than <- cumsum(table(mpg_sorted))
freq_less_than <- rev(cumsum(rev(table(mpg_sorted))))

# Plot the cumulative frequency curves
plot(names(freq_more_than), freq_more_than, type = "l", xlab = "mpg", ylab = "Cumulative Frequency", main = "Cumulative Frequency of More & Less mpg")
lines(names(freq_less_than), freq_less_than, type = "l", col = "red")

#Locate the point of intersection and draw the perpendicular line to the x-axis
intersection_index <- which.min(abs(freq_more_than - freq_less_than))
median_mpg <- names(freq_more_than)[intersection_index]
abline(v=median_mpg, col="green")

Here we locate the intersection point and plot in graph.

Now comparing the median values obtained from graphs and built in function and they are same

# 10. Write advantage and limitations of plots created using R base packages based on these nine steps

Advantages of plots created using R base packages:

  • Familiarity: R base graphics are part of the R language itself, so they are available by default and dont require any additional packages to be installed. Many people are familiar with the syntax and functionality of R base graphics, so they can be a good choice for quick and simple data visualizations.

  • Flexibility: R base graphics are highly customizable and can be used to create a wide variety of plots, including scatterplots, line plots, bar plots, and histograms. You can control many aspects of the plot, such as the axes, labels, colors, and legends.

  • Integration: R base graphics can be easily integrated with other R functions and packages, which can be useful for data analysis and reporting. For example, you can create a plot using R base graphics and then use the summary() function to generate descriptive statistics for the same data.

Limitations of plots created using R base packages:

  • Limited interactivity: R base graphics do not support interactive features such as zooming, panning, and hover-over tooltips. This can make it difficult to explore complex datasets and identify patterns or outliers.

  • Limited aesthetics: While R base graphics are highly customizable, they may not have the same aesthetic appeal as plots created using other packages or tools. For example, some users may find the default colors or fonts used in R base graphics to be unappealing or difficult to read.

  • Steep learning curve: While R base graphics can be powerful and flexible, they can also be challenging to learn and use effectively. The syntax and terminology used in R base graphics can be unfamiliar to new users, and creating complex plots may require a significant amount of trial and error. This can be a barrier to entry for some users who are new to R or data visualization in general.
